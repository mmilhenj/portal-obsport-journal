# portal-obsport-journal

## 项目简介

`portal-obsport-journal` 是一个用于归档和发布独立 HTML 页面的仓库。该项目不针对任何特定域名或网站，仅作为一组自包含页面的存储与展示空间。仓库中的页面内容保持独立，便于索引、回顾与分发。

## 目录结构

```
portal-obsport-journal/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 可选的入口页面（用于导航或索引）
└── README.md       # 本文件
```

各页面之间可共享 `assets/` 目录中的资源，但建议保持每个 HTML 文件的相对独立性，以降低维护耦合。

## 页面归档说明

- 所有归档页面均为标准 HTML 格式，编码统一为 UTF-8。
- 页面文件名建议使用英文小写与连字符，便于 URL 引用与版本管理。
- 每个页面应包含基本的 HTML 结构（`<!DOCTYPE html>`、`<head>`、`<body>` 等）。
- 如需引用外部资源，请尽量使用相对路径，并确保资源已放置于 `assets/` 目录下。
- 仓库不限制页面内容类型，但请遵守法律法规与平台规范。

## 维护说明

- 新增页面时，请在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html`（如有）的链接。
- 修改现有页面时，请通过 Pull Request 提交，并附带简要说明。
- 不建议直接在 `main` 分支上编辑，请使用功能分支进行开发。
- 仓库维护者会定期检查资源完整性，并清理长期未使用的文件。

## 使用方式

你可以通过以下方式使用本仓库的内容：

1. 直接浏览 `pages/` 目录下的 HTML 文件（通过 GitHub Pages 或本地打开）。
2. 克隆仓库到本地，在浏览器中打开任意页面。
3. 将页面内容嵌入其他项目（请保留原始来源信息）。
